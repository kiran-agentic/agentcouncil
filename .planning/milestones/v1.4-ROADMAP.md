## Overview

Unified all backend providers under a single OutsideProvider pipeline with capability metadata, session strategy routing, and skill adaptation. Claude is now the default backend (works out of the box). Legacy `outside_query` tool preserved as deprecated shim.

## Milestone Summary

**Key Decisions:**
- CodexProvider uses `codex mcp-server` (persistent MCP), NOT `codex exec` (one-shot)
- ClaudeProvider uses `claude --print --session-id` (resumed one-shot), NOT persistent stdin
- Provider capabilities declared as metadata (session_strategy, workspace_access, supports_runtime_tools)
- OutsideSession persistent mode sends only latest prompt; replay mode sends full history
- Legacy outside_query routes through provider pipeline with deprecation warning
- Auto-fallback to ClaudeProvider when no backend configured
- Built-in provider strings recognized by ProfileLoader.resolve() for correct dispatch

**Issues Resolved:**
- ProfileLoader string dispatch for built-in provider names (found during audit, fixed in 1f3420d)
- Stale docstrings referencing "codex" as default (fixed to "claude")

- AgentAdapter deprecation removal (deferred to v1.5)
