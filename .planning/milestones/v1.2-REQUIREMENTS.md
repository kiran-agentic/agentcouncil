# Requirements: AgentCouncil v1.2

**Defined:** 2026-04-12
**Core Value:** Independence before convergence

## v1.2 Requirements

### Config System

- [x] **CFG-01**: User can define named backend profiles in `.agentcouncil.json` with provider, model, endpoint, and api_key_env fields
- [x] **CFG-02**: Config precedence resolves correctly: skill arg > project config > global config > env var > default ("codex")
- [x] **CFG-03**: `ProfileLoader` uses pydantic-settings `JsonConfigSettingsSource` for type-safe config parsing
- [x] **CFG-04**: Legacy `AGENTCOUNCIL_OUTSIDE_AGENT` env var preserved as override path (backward compatible)
- [x] **CFG-05**: `show-effective-config` reports each resolved value and its source level
- [x] **CFG-06**: API keys stored as env var names only — never raw secrets in config files

### Provider Architecture

- [x] **PROV-01**: `OutsideProvider` ABC defines `chat_complete(messages, tools?) -> ProviderResponse` contract
- [x] **PROV-02**: `OllamaProvider` connects via official `ollama` SDK AsyncClient with `stream=False` for tool-calling sessions
- [x] **PROV-03**: `OpenRouterProvider` connects via `openai` SDK with `base_url` swap and Bearer token auth
- [x] **PROV-04**: `BedrockProvider` connects via boto3 `converse()` API with `asyncio.to_thread()` wrapping
- [x] **PROV-05**: `BedrockProvider` normalizes `toolUse`/`toolResult` naming to internal format
- [x] **PROV-06**: Each provider implements `auth_check()` with actionable error messages before first deliberation call
- [x] **PROV-07**: `BedrockProvider` creates boto3 client per-invocation to avoid STS credential expiry mid-session
- [x] **PROV-08**: Backend SDKs are optional extras in pyproject.toml (Ollama-only users don't pull boto3)

### Deliberation Runtime

- [x] **RUN-01**: `OutsideRuntime` executes read-only tool loop with locked tool surface: `list_files`, `search_repo`, `read_file`, `read_diff`
- [x] **RUN-02**: All file paths from LLM tool calls validated via `os.path.realpath()` + workspace prefix check (path traversal prevention)
- [x] **RUN-03**: Extension allowlist rejects `.env`, `.pem`, `*.key` and other sensitive file patterns
- [x] **RUN-04**: Per-turn token budget enforced to prevent O(n^2) transcript growth with stateless replay
- [x] **RUN-05**: Tool-call retry limit of 3 per turn with graceful degradation to text-only mode
- [x] **RUN-06**: `OutsideRuntime` handles both native function-calling and textual action protocol (`READ_FILE path=...`)
- [x] **RUN-07**: Textual tool protocol opt-in via conformance certification (not enabled by default for uncertified models)

### Session Layer

- [x] **SESS-01**: `OutsideSession` composes Provider + Runtime into protocol contract with open/call/close lifecycle
- [x] **SESS-02**: `OutsideSessionAdapter` wraps `OutsideSession` behind existing `AgentAdapter` interface for backward compatibility
- [x] **SESS-03**: Transcript replay for stateless backends accumulates and resends full context each round
- [x] **SESS-04**: `AgentAdapter` deprecated with import-level `DeprecationWarning` — no new providers implement it

### MCP Integration

- [x] **MCP-01**: `outside_start(prompt, profile?, model?)` MCP tool returns session_id and first response
- [x] **MCP-02**: `outside_reply(session_id, prompt)` MCP tool returns next response
- [x] **MCP-03**: `outside_close(session_id)` MCP tool cleans up session resources
- [x] **MCP-04**: `get_outside_backend_info(profile?, model?)` MCP tool returns capabilities without opening a session
- [x] **MCP-05**: Existing 4 MCP tools (brainstorm/review/decide/challenge) migrated to use ProfileLoader + OutsideSession

### Skill Updates

- [x] **SKIL-01**: All 4 skills accept optional `backend=` argument for per-invocation backend selection
- [x] **SKIL-02**: Skills use uniform MCP session tools (outside_start/reply/close) — no backend-specific branching
- [x] **SKIL-03**: Skills inspect `workspace_access` capability via `get_outside_backend_info` to decide context strategy

### Conformance Certification

- [x] **CERT-01**: `ConformanceCertifier` validates backend capability: tool use, multi-turn coherence, structured output, budget adherence
- [x] **CERT-02**: Certification results cached in backend profile keyed by `(model_id, provider_version, agentcouncil_version)`
- [x] **CERT-03**: Stale certification triggers warning with re-certification command
- [x] **CERT-04**: Protocol compatibility gates: prompt-only models blocked from review/challenge, degraded models warned

### Transcript Metadata

- [x] **META-01**: Transcript model includes `outside_provider`, `outside_profile`, `outside_model` fields
- [x] **META-02**: Transcript model includes `outside_session_mode` (native_persistent, replay) and `outside_workspace_access` (native, assisted, none)
- [x] **META-03**: All 4 protocol engines populate metadata fields in returned transcripts

### Documentation

- [x] **DOC-01**: PROTOCOLS.md with side-by-side matrix comparing all 4 protocols
- [x] **DOC-02**: BACKENDS.md documents provider setup, independence tiers, and capability differences
- [x] **DOC-03**: ARCHITECTURE.md documents OutsideProvider/Runtime/Session design and migration from AgentAdapter
- [x] **DOC-04**: README.md updated with protocol chooser, install, quickstart, and backend selection

### OSS Packaging

- [x] **OSS-01**: pyproject.toml has complete metadata with optional backend extras
- [x] **OSS-02**: .gitignore excludes __pycache__, .egg-info, .planning, .mcp.json, logs
- [x] **OSS-03**: CONTRIBUTING.md exists with development setup guide
- [x] **OSS-04**: Version aligned across pyproject.toml, server.py, plugin.json

### Testing

- [x] **TEST-01**: Contract tests per provider validate request/response format
- [x] **TEST-02**: Config precedence tests cover all 5 levels and legacy env var
- [x] **TEST-03**: Path traversal prevention tests including symlink escape
- [x] **TEST-04**: Tool-loop retry and degradation tests with StubProvider
- [x] **TEST-05**: Transcript metadata populated correctly for each provider type
- [x] **TEST-06**: Existing 273+ tests continue passing (backward compatibility)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Full general-purpose agent runtime | Deliberation harness only — read-only, no shell, no writes (brainstorm consensus) |
| Heuristic model quality taxonomy | Skills stay backend-agnostic; no model_class hints (brainstorm consensus) |
| Write/execute tools for outside models | Read-only workspace access only (brainstorm consensus) |
| Auto-routing between backends | Conflicts with product identity — user explicitly chooses |
| Config file for secrets | Only env var names stored; never raw API keys (security requirement) |
| Streaming + tool calling simultaneously | Ollama limitation makes this non-universal; defer |
