### 🚧 v1.3 Kiro CLI Provider (In Progress)

**Milestone Goal:** Enable the Kiro CLI as an outside agent backend that works seamlessly like Codex — persistent subprocess, full deliberation participation, same user experience.

## Phase Details

### Phase 18: KiroProvider Implementation
**Goal**: Kiro CLI participates in deliberations as a fully integrated outside agent backend
**Depends on**: Phase 17
**Requirements**: KCFG-01, KCFG-02, KIRO-01, KIRO-02, KIRO-03, KIRO-04, KIRO-05, KIRO-06
**Success Criteria** (what must be TRUE):
  1. A user can add `provider: "kiro"` to `.agentcouncil.json` and run a brainstorm with Kiro as the outside agent — the same way they would with Codex
  2. When Kiro's two-process tree (kiro-cli + kiro-cli-chat) is running, closing the session terminates both processes cleanly via staged SIGTERM
  3. Starting a session without Kiro credentials present raises a ProviderError with an actionable message telling the user to run `kiro-cli auth login`
  4. The ACP session lifecycle (initialize → session/new → session/prompt → AgentMessageChunk accumulation → TurnEnd) completes a full deliberation round without error
  5. Permission requests from Kiro during a session are automatically denied with a structured JSON-RPC error, and the session continues uninterrupted
**Plans**: 2 plans

Plans:
- [x] 18-01: BackendProfile extension (cli_path, auth_token_env) + _make_provider() Kiro dispatch
- [x] 18-02: KiroProvider core — ACP subprocess, session lifecycle, permission denial, history corruption handling

### Phase 19: Contract Tests + Backward Compatibility
**Goal**: KiroProvider behavior is verified by mock-based tests and existing tests continue passing
**Depends on**: Phase 18
**Requirements**: KTEST-01, KTEST-02, KTEST-03
**Success Criteria** (what must be TRUE):
  1. Mock-based ACP tests verify the correct JSON-RPC request/response sequence (initialize → session/new → session/prompt) without requiring a real Kiro CLI binary
  2. Two-process teardown tests verify that both kiro-cli and kiro-cli-chat child processes receive SIGTERM in the correct order using mock subprocess
  3. All 458+ existing tests continue passing after KiroProvider is introduced — no regressions in Ollama, OpenRouter, Bedrock, Codex, or Claude paths
**Plans**: 2 plans

Plans:
- [x] 19-01: ACP contract tests and two-process teardown tests
- [x] 19-02: Full test suite backward compatibility validation

### Phase 20: Documentation
**Goal**: Users know how to set up and use the Kiro backend, including its headless limitations
**Depends on**: Phase 19
**Requirements**: KDOC-01, KDOC-02
**Success Criteria** (what must be TRUE):
  1. BACKENDS.md contains a Kiro section with setup steps, auth prerequisites, the headless limitation (no KIRO_AUTH_TOKEN yet), and a clear workaround note
  2. README.md backend comparison table includes Kiro alongside Codex, Claude, Ollama, OpenRouter, and Bedrock
**Plans**: 1 plan

Plans:
- [x] 20-01-PLAN.md — Add Kiro to BACKENDS.md and README.md backend comparison table
