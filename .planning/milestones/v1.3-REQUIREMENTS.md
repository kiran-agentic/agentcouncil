# Requirements: AgentCouncil v1.3

**Defined:** 2026-04-13
**Core Value:** Independence before convergence

## v1.3 Requirements

### Kiro Provider

- [x] **KIRO-01**: `KiroProvider` implements `OutsideProvider` via `kiro-cli acp` JSON-RPC 2.0 subprocess with `--trust-all-tools` flag
- [x] **KIRO-02**: Process tree management handles two-process architecture (kiro-cli + kiro-cli-chat) with explicit child enumeration via `pgrep -P` and staged SIGTERM cleanup
- [x] **KIRO-03**: `auth_check()` detects missing credentials via file-system pre-flight check of `~/.kiro/` tokens and raises `ProviderError` with actionable "run kiro-cli auth login" message
- [x] **KIRO-04**: ACP session lifecycle handles `initialize` → `session/new` → `session/prompt` → `AgentMessageChunk` accumulation → `TurnEnd` correctly
- [x] **KIRO-05**: Permission requests from Kiro (`session/request_permission`) are detected by the reader loop and immediately denied with a structured JSON-RPC error reply
- [x] **KIRO-06**: Session history corruption (upstream issue #6110) detected via `"invalid conversation history"` error and handled as session-restart signal

### Config Integration

- [x] **KCFG-01**: `BackendProfile` extended with optional `cli_path` (binary override) and `auth_token_env` (future KIRO_AUTH_TOKEN) fields
- [x] **KCFG-02**: `_make_provider()` factory in `server.py` dispatches `provider: "kiro"` to `KiroProvider` with lazy import

### Testing

- [x] **KTEST-01**: Mock-based contract tests validate ACP JSON-RPC request/response format without a real Kiro CLI
- [x] **KTEST-02**: Two-process teardown tests verify child process cleanup using mock subprocess
- [x] **KTEST-03**: Existing 458+ tests continue passing (backward compatibility)

### Documentation

- [x] **KDOC-01**: BACKENDS.md updated with Kiro provider setup, auth prerequisites, and headless limitation prominently documented
- [x] **KDOC-02**: README.md backend comparison table includes Kiro

## v1.4+ Requirements (Deferred)

### Future

- **FUT-01**: Migrate Codex/Claude CLI adapters to OutsideProvider (unify legacy + new paths)
- **FUT-02**: Remove AgentAdapter entirely once all backends use OutsideProvider
- **FUT-03**: KIRO_AUTH_TOKEN env var support (pending upstream kirodotdev/Kiro#5358)
- **FUT-04**: Additional providers (Groq, Fireworks, Azure OpenAI, vLLM)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Headless CI auth for Kiro | Upstream limitation — no KIRO_AUTH_TOKEN exists (issue #5938) |
| Tool injection into Kiro's tool loop | Kiro manages its own tools; OutsideRuntime's read-only harness does not apply |
| Streaming responses from Kiro | AgentMessageChunk accumulation is handled internally; no streaming API exposed to callers |
| Codex/Claude migration to OutsideProvider | Separate milestone — changes scope and risk profile significantly |
