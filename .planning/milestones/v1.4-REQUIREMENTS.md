# Requirements: v1.4 Unified Provider Pipeline

**Defined:** 2026-04-13
**Core Value:** Independence before convergence

## Provider Unification

- [x] **UPROV-01**: CodexProvider implements OutsideProvider using `codex mcp-server` for persistent MCP session management
- [x] **UPROV-02**: ClaudeProvider implements OutsideProvider using `claude --print` with `--session-id` for resumed one-shot sessions
- [x] **UPROV-03**: `_make_provider()` dispatches `provider: "codex"` to CodexProvider and `provider: "claude"` to ClaudeProvider
- [x] **UPROV-04**: Each OutsideProvider declares capability metadata: `session_strategy` (persistent/replay), `workspace_access` (native/assisted/none), `supports_runtime_tools` (bool)

## Skill Unification

- [x] **USKIL-01**: All 4 skill files (brainstorm, review, decide, challenge) use only `outside_start`/`outside_reply`/`outside_close` — no direct `mcp__codex__codex` or `mcp__agentcouncil__outside_query` calls
- [x] **USKIL-02**: Skills adapt prompt construction based on provider capabilities — include file contents for no-workspace backends, skip for native-workspace backends
- [x] **USKIL-03**: `allowed-tools` in skill frontmatter lists only AgentCouncil session tools (`outside_start`, `outside_reply`, `outside_close`, `get_outside_backend_info`)

## Session Mode

- [x] **USESS-01**: OutsideSession sends only the latest user prompt to persistent providers (codex, claude, kiro) instead of replaying full history
- [x] **USESS-02**: OutsideSession replays full history for replay providers (ollama, openrouter, bedrock) — existing behavior preserved
- [x] **USESS-03**: `get_outside_backend_info` returns provider capabilities (session_strategy, workspace_access, supports_runtime_tools) so skills can adapt

## Auto-Fallback

- [x] **UFALL-01**: When no backend is configured anywhere (no profile, no env var, no default_profile), AgentCouncil defaults to ClaudeProvider
- [x] **UFALL-02**: Explicit `backend=codex` or `provider: "codex"` in config fails with actionable error if codex binary is missing — no silent fallback

## Backward Compatibility

- [x] **UCOMPAT-01**: `outside_query` MCP tool continues to work, internally routed through the provider pipeline, with a deprecation warning
- [x] **UCOMPAT-02**: All 493+ existing tests continue passing after provider and skill changes

## Documentation

- [x] **UDOC-01**: BACKENDS.md, ARCHITECTURE.md, and README.md updated to reflect the unified provider pipeline

## Future Requirements (v1.5)

- [ ] **FUT-01**: Remove legacy adapters (ClaudeAdapter, CodexAdapter, StubAdapter) entirely
- [ ] **FUT-02**: Remove `outside_query` MCP tool
- [ ] **FUT-03**: Remove `resolve_outside_adapter()` and `resolve_outside_backend()` functions
